## ⚡ Особенности проектирования и архитектурные вызовы
В рамках этого кейса я решала не просто задачу отрисовки схем, а прорабатывала сложные технические вызовы для высоконагруженных систем:

1.  **Борьба с Race Condition (гонкой за ресурсами):** Проектирование транзакционных блокировок строк на уровне СУБД с использованием `SELECT FOR UPDATE` для защиты от параллельных списаний.
2.  **Отказоустойчивость контура (Fault Tolerance):** Проектирование асинхронной архитектуры для защиты данных при падении внешнего шлюза Центробанка.
3.  **Защита от дубликатов транзакций:** Внедрение паттерна **Идемпотентности** (`Idempotency-Key` в Headers) для обработки повторных кликов при нестабильной связи.

## 🛠️ Что внутри полного кейса (Стек & Артефакты):
*   **Архитектура:** Распределенная микросервисная модель, асинхронное взаимодействие через брокеры сообщений.
*   **Моделирование:** Проектирование процессов в нотациях **BPMN 2.0** и **UML** (Sequence, State Machine).
*   **Интеграции:** Проектирование гибридных контрактов — **REST API (JSON)** для мобильного приложения и **SOAP (XML ISO 20022)** для шлюза ЦБ.
*   **Базы данных:** Разработка оптимизированных SQL-запросов (`SELECT EXISTS`) под высокую нагрузку.
